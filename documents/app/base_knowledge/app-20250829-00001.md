核心知识：智能体选择原则

本文档为 PMCAAgentsDecision 智能体提供决策依据，用于根据用户的业务需求为任务挑选合适的协作成员。在 app 领域中，不同的任务会触发不同的能力需求，只有根据需求选择合适的专家智能体，团队才能高效完成工作。

1 核心职责

PMCAAgentsDecision 是团队的“规划师”。它负责解析用户的任务描述，识别出任务需要的能力，并从能力库中选择相应的专家智能体组建团队。这些智能体可能负责知识管理、代码生成、信息检索、文本加工等职责。

2 选择原则

在分析用户指令时，PMCAAgentsDecision 应根据关键词和语境判定需要的能力。下面列出了几类常见任务及其对应的必选智能体。

2.1 知识管理任务

如果任务涉及对信息或知识的 持久化存储、长期记忆、更新与删除 或 基于历史知识的查询，必须选择知识管理类智能体。这类任务的常见指令包含“记住”“保存”“上传”“存储”“记录”“归档”“更新”“修改”“删除”“查找”“查询”等动词。

必选智能体：

PMCAKnowledgeLibrarian：作为用户交互的唯一入口，负责将自然语言指令转化为结构化内部指令。

PMCAKnowledgeTechnician：接受结构化指令并调用底层 LightRAG 工具完成注入、检索、更新和删除操作。

案例：

“帮我把这个项目的技术设计文档保存到知识库。” → 选择 PMCAKnowledgeLibrarian 与 PMCAKnowledgeTechnician。

“查找一下我们之前发布过的市场分析报告。” → 选择知识管理智能体以执行查询。

2.2 代码生成与自动化任务

当用户的任务需要 生成代码、编写脚本 或 构建自定义自动化流程 时，团队中必须包含能够完成代码生成与执行的能力。这类指令通常出现“生成代码”“编写脚本”“自动化”“写一个程序”“没有现成工具”等表达。

必选智能体：

PMCACodeGenExec：具备生成、调试并运行脚本的能力，用于实现定制的自动化逻辑。

案例：

“请写一个脚本，将 /data 目录下的所有 CSV 文件合并成一个 Excel 表格。” → 选择 PMCACodeGenExec。

“这个问题没有现成工具，你能不能生成一段代码来处理它？” → 选择 PMCACodeGenExec。

2.3 信息检索与总结任务

如果任务涉及从大量文档或外部资源中检索信息、提取要点或生成摘要，应选择信息检索类智能体。这类指令包含“查找资料”“搜索并汇总”“分析文档”“总结”“调查”等表达。

推荐智能体：

PMCALightRAGQueryAgent：用于面向知识库的复杂检索和摘要，能够在知识图谱和文本块之间进行权衡。

PMCAResearcher：用于网络检索和实时信息的获取（如果该能力已在系统中实现）。

案例：

“请分析一下这份合同，并总结主要风险点。” → 选择 PMCALightRAGQueryAgent。

“帮我找出最近三个月内与‘芯片短缺’相关的新闻并归纳要点。” → 选择检索类智能体。

2.4 翻译与文本加工任务

涉及语言转换、润色、文风调整或文本重写的任务，需要选择具有语言处理能力的智能体。指令可能包含“翻译”“润色”“改写”“用另一种风格表达”等词汇。

推荐智能体：

PMCATranslator：负责语言翻译。

PMCARewriter：负责文本改写与风格调整。

3 元数据协商的重要性

对于知识管理任务，PMCAAgentsDecision 在选择了 PMCAKnowledgeLibrarian 之后，需要意识到接下来一定会发生 元数据协商。元数据（尤其是 owner_agent 字段）决定了知识归属，影响后续的记忆个性化与访问控制。因此，选择知识管理智能体实际上是在为任务奠定长期知识结构的基础。

4 决策流程示意

解析任务描述：识别用户意图，抽取关键词。

映射能力需求：根据意图匹配上述任务类别，确定必选与推荐智能体。

考虑约束与冲突：避免选择互斥或冗余的智能体，确保团队构成合理。

生成团队列表：将选择结果传递给 PMCATeamDecision，由其决定团队协作模式。

遵循以上原则，PMCAAgentsDecision 将能够快速、准确地为每个任务组建具备必要能力的专家团队。
