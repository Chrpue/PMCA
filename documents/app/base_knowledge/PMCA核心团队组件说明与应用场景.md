# PMCA核心团队组件说明与应用场景

本文档是`PMCATeamDecision`智能体进行决策的核心依据。

### 1. `RoundRobin` (轮询模式)

- **适用场景**: 线性流水线任务、简单序列对话、提议-批判循环。
- **优势**: 简单可控，流程确定。
- **劣势**: 僵化，效率可能低下。
- **决策案例**: “对一个已有的代码方案进行审查”时，非常适合构建`提议者`和`批判者`的两人小组。

### 2. `Swarm` (蜂群模式)

- **适用场景**: 探索性任务、需要动态规划、需要局部专家决策。
- **优势**: 高度灵活，效率高。
- **劣势**: 难以控制，对智能体判断力要求高。
- **决策案例**: “分析一个紧急的线上Bug”，允许`日志分析师`、`代码审查员`、`代码修复师`之间灵活切换。

### 3. `GraphFlow` (图流程模式)

- **适用场景**: 复杂的、需要稳定复现的业务流程（BPM），有明确分支和依赖的任务。
- **优势**: 控制力最强，高度可靠，支持并行。
- **劣势**: 设计复杂，前期成本高，灵活性差。
- **决策案例**: “处理一笔新的用户订单”，可以完美定义`验证库存->扣减库存->支付->更新状态`的流程。

### 4. `MagenticOne` (磁力一号模式)

- **适用场景**: 需要与外部世界（文件系统、网页）交互的开放性问题。
- **优势**: 强大的外部交互能力，自带规划与协调核心。
- **劣势**: 专用性强，可能过于“重”，存在中心化瓶颈。
- **决策案例**: “调研并总结AutoML领域的最新五篇顶会论文”，协调员会自动规划搜索、下载、总结的完整流程。
