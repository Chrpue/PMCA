# 核心知识：智能体选择的批判性原则（反模式）

本文档为 `PMCAAgentsDecisionCritic` 智能体提供核心的审计依据，用于发现并修正团队构成方案中的明显缺陷。

### 1. 核心反模式：缺少代码生成能力

- **情景描述**: 当用户的任务明确要求通过**“生成代码”**、**“编写脚本”**或**“执行一个自定义的自动化流程”**来解决一个没有现成工具的问题时，团队构成方案中**必须包含 `PMCACodeGenExec` 智能体**。
- **审计关键点**: 如果在一个需要动态代码能力的任务中，`PMCAAgentsDecision` 提出的团队成员列表里遗漏了 `PMCACodeGenExec`，那么这个方案就是**有严重缺陷的**。
- **触发指令示例**:
  - _“请帮我写一个 Python 脚本，自动整理 `/data` 目录下的所有 CSV 文件。”_
  - _“我需要一个自动化的方式来监控某个网站的更新，你能帮我实现吗？”_
  - _“这个问题没有现成的工具，你能不能生成一段代码来处理它？”_

### 2. 批判与修正建议

当检测到上述反模式时，`PMCAAgentsDecisionCritic` 必须提出修正建议。

- **标准回应**:
  - **决策**: `[AGENTS DECISION REVISE]`
  - **原因**: “任务的核心需求是生成自定义代码来解决问题，但当前的团队构成中缺少具备代码生成与执行能力的 `PMCACodeGenExec` 智能体。这是一个关键的能力缺失，将导致任务无法完成。建议立即将其增补入团队。”

### 3. 战略重要性

`PMCACodeGenExec` 是 PMCA 系统**动态能力**和**问题解决能力上限**的保障。它使得团队能够应对那些超出预定义工具范围的、更加开放和复杂的任务。因此，在审计时，对 `PMCACodeGenExec` 的需求识别必须具有最高的敏感度。
