# 智能体选择策略：知识管理

本文档为`PMCAAgentsDecision`智能体提供核心决策依据，用于判断何时应将知识管理相关的智能体纳入任务团队。

### 1. 核心原则

当用户的任务指令涉及到对信息或知识的**“持久化存储”**、**“长期记忆”**、**“查询与检索”**或**“结构化管理”**时，**必须**将 `KnowledgeLibrarian` 选入协作团队。`KnowledgeLibrarian` 是所有知识管理任务的统一入口。

### 2. 触发场景与案例分析

- **存储意图**: 指令包含“记住”、“保存”、“上传”、“存储”、“记录”、“归档”等。
  - _案例_: “帮我把这份会议纪要存起来。” -> **必须选择 `KnowledgeLibrarian`**。
- **查询意图**: 指令包含“查找”、“搜索”、“查询”、“调取”，且指向非实时信息。
  - _案例_: “查找一下我们关于‘苍穹计划’的所有技术文档。” -> **必须选择 `KnowledgeLibrarian`**。
- **管理意图**: 指令涉及对知识的维护，如“更新”、“删除”。
  - _案例_: “更新‘PMCA框架’的介绍文档。” -> **必须选择 `KnowledgeLibrarian`**。

### 3. 元数据的重要性（关键决策点）

选择 `KnowledgeLibrarian` 的核心是为了给知识赋予归属。它的核心职责是确定知识的 `owner_agent` 元数据，这是实现“为每一个智能体构建专属记忆网络”的关键。因此，选择`KnowledgeLibrarian`意味着后续流程必然包含与用户协商元数据的环节。
